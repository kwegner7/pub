COMPILE IN THE WORK AREA

There are four files that result from the compilation process:

k:\work\tools\GENFIT.exe
k:\work\tools\GENFIT.dll
k:\work\tools\genfit_dose.dll
k:\work\tools\xerces-c_3_2D.dll

These files may be transferred from outside to inside the ADTEC and they will function correctly without re-compiling. There may be a reason to change the source code, for example, to insert a print statement. (There is no reason to change the Xerces source code.) The following Powershell commands will re-compile and regenerate the exe and dlls in the tools folder:

cd k:\work
tools\build

MSBuild

The build.ps1 script invokes the Visual Studio project by means of the MSBuild program. Therefore the location of MSBuild.exe must be in the Powershell path. The Powershell path can be set in the profile script. If it does not exist, create the file:

C:\Users\<your name>\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1

Add the following lines to the user Powershell script (the location of the MSBuild may be different among the Windows 10 machines in the ADTEC):

$env:Path += ";C:\Program Files (x86)\MSBuild\14.0\Bin"
$env:Path += ";C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\bin"

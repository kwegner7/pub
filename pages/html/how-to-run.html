HOW TO RUN RADIATION EFFECTS SOFTWARE

Each run is determined by an input xml file. And you may specify where you want the outputs to go. If not specified, the outputs will go to the current working directory. In the k:\work\run folder are sub-folders called inputs and outputs. But you do not have to use these folders, they are provided for convenience. You may specify absolute paths or relative paths. The following are all equivalent ways to process the input file called envrng.xml and the outputs go to the folder called outputs:

cd k:\work
tools\run run\inputs\envrng.xml run\outputs

cd k:\work
tools\run -FileIn run\inputs\envrng.xml -FolderOut run\outputs

cd k:\work\tools
run k:\work\run\inputs\envrng.xml ..\run\outputs

cd k:\work\run\outputs
k:\work\tools\run ..\inputs\envrng.xml

cd to anywhere
k:\work\tools\run k:\work\run\inputs\envrng.xml k:\work\run\outputs

In the folder k:\work\run\outputs will now be the files:

envrng-out.xml
envrng.csv

You can run all input xml files within a folder. The following are all equivalent ways to process a folder:

cd k:\work
tools\run-all run\inputs run\outputs

cd k:\work
tools\run-all -FolderIn run\inputs -FolderOut run\outputs

cd to anywhere
k:\work\tools\run-all k:\work\run\inputs k:\work\run\outputs

INPUT XML FILE

Radiation Effects Software can determine:

-- dose value given positions of the burst and receiver
-- distance between burst and receiver that acheives a constraint dose
-- yield of device that acheives a constraint dose

The input xml file determines:

-- characteristics of the device
-- sequence of positions of the burst and receiver
-- environments of interest

Details of how to configure the input xml file are contained in the User Manual

Detailed software requirements are contained in the Software Requirements Specification

USING CSV FILE TO SPECIFY THE GEOMETRY

When the run kind is "FLYTHRU" the program will read the geometry from a CSV file. The columns of the CSV file are:

altitude of burst in feet
altitude of receiver in feet
distance between burst and receiver in feet

The output CSV file will have the following columns:

altitude of burst in feet
altitude of receiver in feet
distance between burst and receiver in feet
angle between burst and receiver in degrees
horizontal offset between burst and receiver in feet
air depth (rohr) in g/cm2
dose value

The location of the CSV file is specified in the input xml file under the <geometry> and <runkind> tags. The other sub-tags of <geometry> are ignored. For example:

<geometry>
  <runkind>FLYTHRU K:\\work\\runs\\geometry\\HeightHeightSlant.csv<runkind>
    . . .
</geometry>
